# Contributing to PM-OS

Thanks for your interest in contributing!

## How to Contribute

1. **Open an issue first** — Discuss your idea before coding
2. **Fork the repo** — Make changes in your fork
3. **Follow conventions** — Match existing code style
4. **Write clear commits** — Use imperative mood ("Add feature", not "Added feature")
5. **Test locally** — Ensure everything works
6. **Submit PR** — Reference the issue number

## Areas for Contribution

- 📚 Documentation improvements
- 🧪 Additional tests
- 🐛 Bug fixes
- 💡 New skill ideas (discuss first)
- 🎨 Examples and tutorials

## Commit Message Style

Good:
```
Add cohort retention visualization

Extends /analyze-cohorts command with visual charts
```

Bad:
```
updated some files
```
